NSI Designer

NSI Designer is a free PySide6 desktop application for creating Windows installers with the Nullsoft Scriptable Install System (NSIS). It provides a graphical interface to generate and compile .nsi scripts without manual scripting. This documentation gives you a quickstart guide and explains all input fields, presets, and advanced options. The motivation for writing the program was to make distributing my Python applications frozen with PyInstaller as convenient as possible through Windows installers.

This section describes all input fields available in the left pane of the GUI. Each field directly maps to values in the generated NSIS script. Understanding their meaning ensures that your installer behaves as expected.

Metadata

Metadata defines the identity of your installer. These values appear in window titles, uninstall entries, and shortcuts. They are also used in the generated script for consistency.

  • Application name ‚Äì used in window titles, uninstall entries, and shortcuts
  • Version ‚Äì included in filenames and uninstall dialogs
  • Company name ‚Äì appears in uninstall information
  • Caption ‚Äì title bar text of the installer window
  • Branding text ‚Äì small text displayed in the installer window
  • About URL ‚Äì (optional) link displayed in the classic Windows uninstall section
  • Help URL ‚Äì (optional) link displayed in the classic Windows uninstall section
  • Update URL ‚Äì (optional) link displayed in the classic Windows uninstall section
  • Comments ‚Äì (optional) a comment displayed in the classic Windows uninstall section
  • Executable file ‚Äì path to the main .exe; all files in the same directory are packaged

Presets

Presets control where and how the installer runs. They determine the default installation directory, the required execution privileges, and whether the installer affects all users or only the current user. Scope and execution level are automatically adjusted based on the installation location.

  • Install location:
    • 64‚Äëbit Program Files ‚Äì installs under C:\Program Files on 64‚Äëbit systems.
    • 32‚Äëbit Program Files ‚Äì installs under C:\Program Files (x86) on 64‚Äëbit systems, or C:\Program Files on 32‚Äëbit systems.
    • Per‚Äëuser AppData ‚Äì installs under the current user‚Äôs %LOCALAPPDATA%, no admin rights required.
  • Execution level:
    • admin ‚Äì installer requests elevation (UAC prompt) and writes to system‚Äëwide locations.
    • user ‚Äì installer runs without elevation, limited to user‚Äëwritable locations.
  • Scope:
    • System‚Äëwide ‚Äì registry and environment changes apply to all users.
    • Per‚Äëuser ‚Äì registry and environment changes apply only to the current user.

Assets

Assets customize the look and feel of the installer. They are optional, but when provided, they enhance branding and user experience.

  • Install icon ‚Äì optional (.ico, .png or .jpg)
  • Uninstall icon ‚Äì optional (.ico, .png or .jpg)
  • Welcome bitmap ‚Äì optional (.bmp, .png or .jpg); if not provided, the welcome page is omitted

Languages

NSI Designer supports a predefined set of languages. English is always included automatically. The language dialog allows you to select additional languages from the available set.

Registry Entries

Registry entries allow you to configure system or application settings during installation. Each entry is mirrored during uninstall to ensure clean removal.

Table columns:

  • Root ‚Äì the top‚Äëlevel registry hive where the entry is stored, e.g.
    • HKLM (HKEY_LOCAL_MACHINE) for system‚Äëwide settings or
    • HKCU (HKEY_CURRENT_USER) for per‚Äëuser settings.
  • Sub‚ÄëKey ‚Äì the path inside the hive, similar to a folder structure, e.g. Software\MyApp.
  • Key‚ÄëName ‚Äì the actual name of the registry value you want to create or modify, e.g. InstallPath.
  • Value ‚Äì the data assigned to the key, e.g. C:\Program Files\MyApp.
  • Type ‚Äì the data type of the value, typically string (REG_SZ) for text or dword (REG_DWORD) for numeric values.

Environment Variables

Environment variables can be set or extended during installation. The uninstaller reverses these changes safely.

Table columns:

  • Name ‚Äì the identifier of the environment variable, e.g. PATH or JAVA_HOME.
  • Value ‚Äì the content to assign or append, e.g. C:\Program Files\MyApp\bin.
  • Mode ‚Äì defines how the variable is handled:
    • set ‚Äì creates a new variable or overwrites an existing one with the given value.
    • append ‚Äì adds the value to the end of an existing variable (commonly used for PATH).

Important: Never use set mode for existing variables like Path, as this would overwrite the entire variable. Use append instead. The latter option automatically creates the environment variable if it does not exist.

Encoding & Export

Encoding determines how the script is saved. Export and compile options let you generate the final installer package.

  • Encoding: ‚Äì UTF‚Äë8 or ANSI (default); ANSI saves the script in the local codepage
  • Export NSIS script and assets ‚Äì Exports the project, .nsi-script and all assets (Icons, License, etc.)
  • Compile last export with NSIS ‚Äì Makensis path is detected automatically or requested if not yet configured

Power User Options ü¶æ

Advanced users and system administrators can control the installer via command‚Äëline flags. These options allow silent installs, logging, and custom installation paths. They are especially useful for automated deployments and scripted setups.

  • /S ‚Äì Silent installation
  • /NOICONS ‚Äì Do not create shortcuts
  • /LOG[=path] ‚Äì Enable logging, optionally specify log file path (Include the filename!)
  • /D=path ‚Äì Set installation directory

Important: The /D=path parameter must always be the last parameter on the command line. The use of quotation marks (") is prohibited, even if the path contains spaces.

Notes & Best Practices üìù

Follow these guidelines to ensure safe and reliable installers. NSIS is powerful, but incorrect settings can cause issues during uninstall or system configuration. Always validate your configuration before distributing installers.

  • Always test installers and uninstallers on a safe system before production use.
  • Registry edits must be handled with care ‚Äì incorrect keys can cause uninstall issues.
  • Environment variables:
    • Use append mode for PATH‚Äëlike variables.
    • Never use set mode for existing variables such as Path!
  • License and welcome pages are optional:
    • License page is only included if an RTF file is selected.
    • Welcome page is only included if a bitmap is selected.
  • Makensis path is requested automatically if not yet configured.
